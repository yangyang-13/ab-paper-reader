# 论文获取时间范围配置说明

## 📅 时间范围配置（方案3实施）

### v2.1版本优化

我们采用了更合理的时间范围默认值：

| 场景 | 时间范围 | 说明 |
|------|---------|------|
| **定时任务** | 1天 | 每日自动抓取，只获取昨天的新论文，避免重复 |
| **手动获取** | 7天（默认） | 用户手动触发，默认获取最近一周，平衡数量和时间 |
| **可选范围** | 1/3/7/14/30天 | 用户可以根据需要灵活选择 |

---

## 🎯 设计理念

### 1. 定时任务：1天
```python
# backend/scheduler.py
papers = self.arxiv_service.fetch_papers(days_back=1, max_results=50)
```

**原因**：
- ✅ 每天自动运行，只需抓取最新的即可
- ✅ 避免重复抓取已处理的论文
- ✅ 减少API调用和处理时间
- ✅ 保持数据新鲜度

**场景**：
- 每天早上9点自动执行
- 获取昨天发布的新论文
- 无需人工干预

---

### 2. 手动获取：7天（默认）

```python
# backend/main.py
days_back: int = Query(7, ge=1, le=90, description="查询过去多少天的论文")
```

**原因**：
- ✅ 平衡论文数量和处理时间
- ✅ 一周是合理的回顾周期
- ✅ 适合首次使用或补充遗漏
- ✅ 7天内论文数量可控（通常10-30篇）

**场景**：
- 首次使用系统
- 定时任务失败需要补抓
- 想查看一周内的所有论文
- 系统重新部署后补充数据

**预估时间**：
- 7天约10-30篇论文
- 处理时间：5-15分钟（v2.1优化后）

---

### 3. 用户可选范围：1/3/7/14/30天

**前端界面**：
```html
<select id="fetchDays">
    <option value="1">最近1天</option>
    <option value="3">最近3天</option>
    <option value="7" selected>最近7天</option>  <!-- 默认 -->
    <option value="14">最近14天</option>
    <option value="30">最近30天</option>
</select>
```

**使用建议**：

#### 最近1天
- 快速获取最新论文
- 适合：每日手动更新
- 预估：1-5篇，1-3分钟

#### 最近3天
- 周末或短期补充
- 适合：错过几天的用户
- 预估：3-10篇，3-5分钟

#### 最近7天（默认）⭐
- 一周回顾
- 适合：常规使用
- 预估：10-30篇，5-15分钟

#### 最近14天
- 两周总结
- 适合：阶段性回顾
- 预估：20-50篇，10-25分钟

#### 最近30天
- 月度全面抓取
- 适合：首次使用或大量补充
- 预估：50-100篇，25-50分钟

## 🔍 技术细节

### 日期过滤逻辑
```python
# 计算截止日期
cutoff_date = datetime.now(timezone.utc) - timedelta(days=days_back)

# 过滤论文
if result.published >= cutoff_date:
    papers.append(paper_data)
```

### 去重逻辑
```python
# 检查是否已存在
existing = db.query(Paper).filter(
    Paper.arxiv_id == paper_data['arxiv_id']
).first()

if existing:
    logger.info(f"Paper already exists: {paper_data['arxiv_id']}")
    continue  # 跳过已存在的论文
```
